# Zehunt Platform Screens

This project now includes a multi-screen platform structure (not only a landing page).

## Route Map
- / : Platform home and positioning
- /login : User sign in screen
- /signup : User account creation screen
- /builders : Builder rankings and discovery
- /builders/[username] : Builder profile with score components, updates, and opportunities
- /products : Product evidence index
- /products/[id] : Product lifecycle and execution evidence
- /feed : Structured build-in-public feed
- /opportunities : Opportunity engine matches
- /admin : Moderation, event monitoring, and score calibration UI

## Why This Matches Platform Direction
- Builder is the primary entity in navigation and screen hierarchy.
- Product pages are modeled as an evidence layer.
- Feed is structured around execution updates, not vanity social posts.
- Opportunities are represented as a dedicated core workflow.
- Admin exists as a first-class system surface for integrity and calibration.
